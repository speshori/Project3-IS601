How the Internet Works



Domain Names

The DNS or Domain Name Server is often considered the phonebook of the internet. It is what maps the domain name to the IP address so browsers can load internet resources. There are 4 DNS servers involved in loading a webpage: DNS recursor - takes the queries from a client machine for what page they want to load from a web browser for example Root nameserver - first step in translating the host name to an IP address. TLD nameserver - this nameserver is the next step in attaining the needed IP address and hosts the last portion of the hostname like the “com” portion Authoritative nameserver - final nameserver that actually returns the IP address for the requested hostname back to the DNS recursor that made the very initial request.

There are two portions to the DNS structure though, they are the authoritative DNS server and the recursive DNS resolver. The recursive DNS resolver responds to the request from the client and finds the DNS record that is stored. It does this by making a series of requests until it hits the authoritative DNS nameserver which actually holds the DNS records. This will complete the request for a given host’s request.
